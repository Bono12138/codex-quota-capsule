# 视觉设计方向

日期：2026-07-13

## 一句话方向

Quota Capsule 的视觉应该靠近现代 Mac 状态控件，远离后台 dashboard 和炫酷大渐变工具。

它应该像一个现代 Mac 状态物：

> 低打扰、半透明、有质感、常驻不烦、扫一眼就知道状态。

## 视觉关键词

- 小。
- 安静。
- 精致。
- 像系统状态控件。
- 有轻微玻璃感，克制、清楚、可读。
- 状态颜色明确，但不刺眼。
- 默认只露出一句人话。

## 是否做 Mac 玻璃特效

答案：**做 Mac-inspired glass，但不要做过度玻璃特效。**

具体含义：

- Mac 本地版可以用接近 `NSVisualEffectView` / Liquid Glass 的半透明材质。
- 背景是轻微 blur + 半透明深浅材质。
- 边缘有非常轻的高光和描边。
- 阴影柔和，不能像网页卡片浮得很夸张。
- 状态色只用作左侧细条、小圆点、细进度线或文字 accent。

不要做：

- 大面积彩色渐变。
- 背景图。
- 强烈 bokeh / orb。
- 拟物电池大图标。
- 四张卡片并排的 dashboard。
- 类似营销 landing page 的视觉。

## 三层界面

### 第一层：默认小胶囊

这是最重要的界面。

用途：

- 常驻桌面。
- 用户不点开也能知道状态。

建议形态：

```text
[绿色点] 够用 · 重置时预计剩 18–24%
```

或：

```text
够用  重置时预计剩 18–24%
```

尺寸建议：

- 高度：28-36 px。
- 宽度：根据文案自适应，约 180-280 px。
- 圆角：接近 capsule，14-18 px。
- 字体：系统字体，数字用 tabular nums。

状态表达：

- 够用：低饱和绿色。
- 偏快：琥珀色。
- 可能不够 / 已用尽：低饱和红色。
- 初步估算 / 数据暂不可用：灰色；必须配文字，不用颜色伪装确定性。

不要在默认胶囊里放：

- 两条完整进度条。
- 周额度。
- provider 列表。
- 设置按钮。
- 大图标。

### 第二层：详情浮层

用户点击或 hover 小胶囊后出现。

用途：

- 解释判断依据。
- 给用户更多数字，但仍然保持克制。

内容：

- 主结论。
- 本周时间进度 vs 本周额度已用。
- 未来 24 小时建议。
- 最近 24 小时实际用量。
- 最近速度区间与可持续速度。
- 重置时预计余量区间或可能提前用完的风险。
- 预测置信原因。
- 周额度重置日期/倒计时，以及上次成功读取/下次自动读取。
- 历史入口。

视觉：

- 宽度约 320-420 px。
- 主要信息不超过 3 个视觉分组；诊断默认折叠。
- 可以使用细进度条，不用大图表。
- 默认不出现复杂历史曲线。

### 第三层：历史/设置窗口

用途：

- 历史快照。
- 主题。
- 通知。
- provider。
- 数据源状态。
- 导出。

视觉：

- 可以比浮层更像 settings/dashboard。
- 但仍然要保持工具感，不做营销页。
- 历史页可以用折线图、时段条、快照列表。

## 和竞品视觉的关系

### QuotaGem

可以学：

- compact 形态。
- 环形图的 glanceable 感觉。
- expanded panel 的层次。
- 展开面板的备用操作必须视觉降级，默认只露出少量主入口；不要把所有低频动作平铺成密集工具栏。

不要学：

- 多 provider 默认密度。
- 背景图 + 强 glass 的整体风格。

### ClaudeBar

可以学：

- 产品完成度。
- 状态 badge。
- 主题能力。
- 让小工具看起来不像脚本。

不要学：

- 大面积紫粉渐变。
- 默认 dashboard 卡片堆叠。

### Codex Quota Viewer

可以学：

- 系统工具的实用感。
- stale / error 的明确状态。

不要学：

- 过重的管理后台感。
- session manager 视觉密度。

## 推荐视觉方案

当前推荐方案叫：

> Quiet Glass Capsule

它采用 Mac 材质感，同时保留独立产品气质。

### 颜色

基础：

- 背景：半透明黑 / 半透明白，跟随系统深浅色。
- 文字：高对比但不纯白刺眼。
- 描边：低透明度白/黑。

状态色：

- 够用：teal / green，低饱和。
- 偏快：amber。
- 可能不够 / 已用尽：coral / red。
- 初步估算 / 数据暂不可用：neutral gray。

### 字体

- 系统字体。
- 数字用 tabular nums。
- 默认胶囊不使用大标题。
- 详情层标题也不要 hero-size。

### 动效

可以做：

- 状态变化时轻微 pulse 一次。
- hover 时轻微提高亮度。
- 详情浮层淡入 + 轻微位移。

不要做：

- 持续闪烁。
- 大幅弹跳。
- 彩色扫光。
- 影响工作注意力的动画。

## 设计红线

- 不能像工程 demo。
- 不能像复杂 dashboard。
- 不能像广告页。
- 不能把默认胶囊做成四个状态卡片。
- 不能让颜色喧宾夺主。
- 不能为了好看牺牲“数据暂不可用 / 数据已过期”的清晰表达。
- 不能把周额度重置和本地数据读取都叫“刷新”。
- 不能隐藏早期估算的不确定性，也不能让用户等待数小时才看到产品价值。

## 当前信息层级

展开态按以下顺序固定：

1. 结果状态与一句结论。
2. 置信原因。
3. 本周时间 / 本周已用。
4. 未来 24 小时建议 / 最近 24 小时。
5. 观察速度与可持续速度。
6. 本周趋势和预测区间。
7. 周额度重置、数据读取时间。
8. 三个主操作；数据诊断折叠在最底部。

## 后续视觉迭代规则

当前实现继续沿用 Quiet Glass Capsule。后续不是再开一个平行视觉方向，而是用确定性状态截图和真实桌面背景验证：

1. 简中、繁中、英文最长文案均不截断。
2. light、dark、busy background 上仍有足够对比。
3. 初步估算、风险和 stale 不依赖颜色才能理解。
4. 新信息必须证明能改善周额度决策，否则留在诊断或设置层。
5. 任何尺寸变化都要复查折叠、展开、停靠、拖动和 resize。

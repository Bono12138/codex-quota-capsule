# Bug Triage 与发布阻塞规则

日期：2026-07-01

## 基本原则

Bug 的唯一执行来源是 GitHub Issues。

这个文档不维护完整 bug 列表，只定义：

- 什么算 P0 / P1 / P2。
- 什么阻塞内测。
- 什么阻塞正式发布。
- issue 应该怎么写。
- 修复完成前必须验证什么。

这样可以避免 Markdown 清单和 GitHub issue 双重维护导致状态过期。

## 优先级定义

### P0: 必须先修

满足任一条件就是 P0：

- 误导用户额度状态。
- 读取失败被显示成安全。
- 额度用完却显示未知。
- 找不到 Codex CLI 但没有可操作诊断。
- app 无法启动、无法退出、频繁卡死。
- 会读取、记录或上传不该碰的敏感数据。
- 会修改 Codex auth、配置或登录状态。
- 破坏主体验：胶囊不可用、菜单栏状态明显错误、刷新逻辑失效。

P0 必须在新功能前处理。

### P1: 本版应处理

满足任一条件通常是 P1：

- 内测用户会高频遇到，但不直接误导额度判断。
- 错误文案不清楚，导致用户不知道下一步怎么修。
- 三语基础文案缺失或严重不自然。
- 反馈入口缺失，影响内测问题收集。
- 主胶囊交互有明显歧义，例如刷新图标像按钮但不能点击。
- 停靠隐藏态残留详情面板、空白底板或大块透明内容。
- 发布文档缺关键路径，导致用户无法按说明安装。

### P2: 可排期

满足任一条件通常是 P2：

- 不影响当前主体验。
- 需要更多产品设计或用户反馈。
- 需要引入较多新状态、新页面或新持久化。
- 主要服务正式发布后的增长或运营。

例子：

- 历史趋势页。
- 通知。
- 自动更新。
- 主题。
- CSV / JSON 导出。
- Chrome 独立版真实 source。

### P3: 暂缓或实验

满足任一条件通常是 P3：

- 与主营体验距离较远。
- 可能增加隐私解释成本。
- 可能把产品推向 dashboard 或复杂监控工具。
- 需要大量平台特定工程。

例子：

- 语音和音效。
- 遥测。
- 多 provider dashboard。
- 敏感高级工具箱：账号、session、config。
- Windows native。

## 内测阻塞项

以下问题存在时不能发给朋友内测：

- Codex CLI 找不到时只显示未知，没有说明原因和修复方向。
- 真实 quota 能读取到 100% 已用，但 UI 没有显示危险。
- 读取失败后清空上次成功状态。
- 启动首轮读取阶段没有显示“读取中”。
- 菜单栏和浮窗显示不同状态。
- 点击折叠胶囊无法展开，或点击后被误判为拖动。
- 靠边后没有进入完整可见的停靠迷你形态。
- 停靠隐藏态下面还显示展开详情或大块内容。
- app 无法退出。
- UI 文字明显截断，导致来源、接口、错误原因不可读。
- 简中、繁中、英文基础界面缺失。
- 任一语言出现点击、缩放、展开、收起或文字截断问题，但没有横向检查另外两种语言。

## 正式发布阻塞项

以下问题存在时不能公开发布：

- 内测阻塞项未解决。
- README / INSTALL 不能让 Codex 用户按步骤完成安装。
- 隐私边界没有写清楚。
- public 仓库包含内部交接、私人上下文或未确认策略。
- 反馈入口缺失。
- public issues 模板没有提醒用户不要粘贴 token、cookie、auth 文件或 session 正文。
- 未说明当前是否签名、公证、是否需要右键打开。

## Issue 写法

Bug issue 必须包含：

- 发生了什么。
- 期望行为。
- 当前显示状态。
- 是否能在终端运行 `codex --version`。
- 是否能运行 `npm run probe:codex:rate-limits`。
- app 错误文案或截图。
- 不含敏感信息的日志。

Feature issue 必须包含：

- 用户问题。
- 为什么现在需要。
- 是否进入默认层、解释层、历史层、设置层、高级层或敏感层。
- 是否影响主胶囊 less is more 原则。
- 是否涉及隐私、上传、通知或账号状态。

## 修复完成标准

P0 / P1 bug 关闭前至少需要：

- 对应自动化测试，或说明为什么无法自动化。
- 本地跑过相关验证命令。
- 如果涉及 UI，提供截图、录屏或人工验收说明。
- 如果涉及 Codex source，确认没有修改 Codex auth、配置或登录状态。
- 如果涉及文案，确认简中、繁中、英文三语同步。
- 如果 bug 先在某一种语言下出现，修复时必须判断它是语言文案问题、布局容量问题还是通用交互问题，并把结论同步到另外两种语言。

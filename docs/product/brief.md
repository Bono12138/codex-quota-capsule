# 产品简报

## 命名

中文名：额度胶囊

英文名：Quota Capsule

公开表达：

> Quota Capsule / 额度胶囊

命名理由：

- “额度”直接命中用户痛点。
- “胶囊”对应桌面小胶囊形态，容易记住。
- 英文名和仓库名一致，方便 GitHub 搜索和英文传播。

## 一句话

Quota Capsule 是一个 Codex-first 的桌面额度小胶囊，把周额度翻译成一句人话：照最近速度能不能撑到刷新，今天还能放心使用多少。

## 定位

第一版服务 Codex 重度用户，尤其是同时跑多个开发任务、经常担心本周额度是否够用的人。

对外传播时不要把它局限成“只给 Codex 的私人工具”。更好的说法是：

- Codex 是首个适配对象。
- 核心模型适用于有额度窗口的 Agent 产品。
- 其他 Agent 社区可以贡献自己的 source adapter。

## 核心价值

核心价值是替用户完成判断，百分比只是判断依据。

用户真正关心的是：

- 现在还能不能继续用？
- 按当前速度能不能撑到刷新？
- 如果可能撑不到，预测区间有多大？
- 如果能撑到，刷新时大概还有多少余量？
- 为保留安全余量，今天还能使用多少？

## 第一版非目标

- 不自动操作上游服务。
- 不做多账号管理。
- 不做手机端。
- 不做复杂历史曲线。
- 不默认上传任何用量数据。
- 不把读取失败伪装成安全状态。

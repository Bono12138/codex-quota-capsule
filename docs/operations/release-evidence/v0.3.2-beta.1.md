## Code review

- Root cause 1: `v0.3.1` stabilized the AppKit status-bar menu, while the recording used a separate nested SwiftUI menu inside the floating panel. The previous endpoint-only check was applied to the wrong surface and could not detect intermediate disappearance.
- Root cause 2: the first-level redesign initially left More Actions reading observable `store.copy` for each menu item. A background refresh publication invalidated the open SwiftUI menu even though its visible language had not changed. Holding a local copy snapshot stable during quota publications, and synchronizing it only when locale copy changes, prevents that lifecycle reset.
- The final design uses two sibling, one-layer menus: Language exposes the three locale actions, while More Actions keeps the remaining tools using stable localized labels. Forecasting, refresh cadence, reset-credit storage, and the AppKit status menu are unchanged.
- Competitor and OpenAI-window research was refreshed from current primary sources. The public positioning now treats Weekly Only as the current reliable product surface, not a promise that an upstream short window can never return.

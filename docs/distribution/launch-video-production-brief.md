# Launch Video Production Brief

日期：2026-07-05

用途：Quota Capsule 首发宣传视频的大纲、口播文案、标题、封面、X 文本和 GitHub README 承接策略。

## 当前推荐名称

中文名：额度胶囊

英文名：Quota Capsule

理由：

- “额度”直接命中痛点。
- “胶囊”对应桌面小胶囊形态，容易记住。
- 英文名和仓库名一致，搜索成本低。

备选：

| 中文名 | 英文名 | 判断 |
| --- | --- | --- |
| 额度胶囊 | Quota Capsule | 首选，清楚、短、和产品形态一致。 |
| 续航胶囊 | Runway Capsule | 强调“能撑多久”，但搜索时不如 quota 直接。 |
| 额度雷达 | Quota Radar | 有监控感，可能显得更重。 |
| 余量灯 | Quota Light | 轻，但不像开发者工具。 |
| 用量哨兵 | Quota Sentinel | 记忆点强，但中文有点硬。 |

对外建议：

> 中文叫“额度胶囊”，英文叫 Quota Capsule。视频里第一次出现时同时说，后面只说“这个小胶囊”。

## 核心判断

这条视频先讲个人痛点，再讲产品功能。它要证明：

> 我买了 Pro，却还是会因为看不懂额度是否够用而收着用。

产品出现之前，观众必须先相信这个行为真实存在。

## 抖音主标题

备选：

## 90-120 秒视频大纲

### 0-5 秒：钩子

目的：用反差抓住人。

口播方向：

> 我已经买了 ChatGPT Pro，但我还是不敢放心用 Codex。

画面：

- 作者出镜，或桌面 + 作者旁白。
- 画面上出现“Pro 用户也会额度焦虑”。

### 5-25 秒：行为循环

目的：让观众看到真实动作。

口播方向：

> 我会反复打开 usage 页面。看一眼，还剩 90%，心里一紧；再看一眼，掉到 80%，我就开始收着用。最离谱的是，很多时候额度没有用完，到刷新前还剩一大截。

画面：

- 反复切到 usage 页面。
- 回到 Codex 工作界面。
- 百分比可以打码或用 mock。

### 25-45 秒：心理机制

目的：把个人行为讲清楚，但不做学术课。

口播方向：

> 后来我发现，我焦虑的点在于：按现在这个速度，还能不能撑到下一次刷新。有限额的东西会一直占注意力。你明明在写代码，脑子里却在算：这样用下去够不够？

画面：

- 额度从 100 到 80 的简单动画或字幕。
- 切到正在写代码的画面。

### 45-75 秒：产品出现

目的：展示 Quota Capsule 的判断价值。

口播方向：

> 所以我做了一个小工具，中文叫“额度胶囊”，英文叫 Quota Capsule。它常驻在 Mac 桌面和菜单栏，把 Codex 的周额度翻译成更直接的状态：够用、注意、可能用完、校准中。我的关注点从百分比变成了两个工作判断：照最近速度能不能撑到刷新，今天还能放心使用多少。

画面：

- 胶囊折叠状态。
- 展开状态。
- 菜单栏状态。
- 状态颜色变化。

### 75-95 秒：差异与开源

目的：解释为什么做这个项目。

口播方向：

> 我也看过一些相关工具，有菜单栏的，有 dashboard 的，有命令行的，也有偏账号管理的。它们都说明这件事有人需要。我做这个版本，是想把默认体验做得更轻：桌面常驻、低打断、一眼知道能不能撑到刷新。

画面：

- 只放竞品名称文字列表。
- 不放第三方截图。

### 95-115 秒：安装方式

目的：把观众带到 GitHub。

口播方向：

> 现在项目已经开源。早期安装方式也比较适合 Codex 用户：打开 GitHub，把 README 里的提示词交给自己的 Codex，让它帮你 clone、测试、构建和启动。敏感内容默认留在本机。

画面：

- GitHub README。
- Codex-assisted 安装提示词。
- 非敏感终端命令。

### 115-125 秒：反馈

目的：收口，给行动。

口播方向：

> 如果你也有类似的额度焦虑，或者你想给其他 agent 做 adapter，欢迎去 GitHub 试用、提 issue、提 PR。也可以通过抖音、X 或邮箱联系我。人多的话，后面我会拉群一起改。

画面：

- GitHub Issues。
- 抖音账号名。
- X 和邮箱。

>
> 项目已经开源。你可以打开 GitHub，把 README 里的提示词交给自己的 Codex，让它帮你在本机安装运行。欢迎试用、提 issue、提 PR。

## GitHub README 承接策略

README 顶部要回答 4 个问题：

1. 这个工具解决什么焦虑。
2. 它怎么把百分比变成判断。
3. 它默认不上传哪些敏感数据。
4. 视频观众下一步怎么安装。

建议首屏顺序：

1. 产品名：Quota Capsule / 额度胶囊。
2. 一句话：A small macOS quota runway capsule for Codex.
3. 痛点：A percentage does not tell you whether you can keep working.
4. 截图或 GIF。
5. Codex-assisted install prompt。
6. Privacy boundary。

## AI 味审查记录

使用 `dbs-ai-check` 自检重点：

- 避免高密度反转句式。
- 避免哲学式升维词。
- 避免段段收束金句。
- 避免把心理学讲成万能解释。
- 保留个人口语和不完全确定的边界。
